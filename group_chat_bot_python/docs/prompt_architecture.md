# Prompt Architecture

## Короткий ответ

Лучший prompt для этого проекта пишется:

- не на отдельном языке программирования;
- а как структурированный текст;
- с хранением в Python-builder'ах, потому что проект сам на Python.

## Почему так

У prompt две разные задачи:

1. Описать поведение модели.
2. Аккуратно подмешать runtime-контекст: имя пользователя, память, stage, настроение, админку и так далее.

Для первой задачи лучше обычный человеческий язык.
Для второй задачи лучше Python, потому что он собирает готовую финальную строку из шаблонов.

Именно поэтому текущая схема такая:

- `app/services/prompt_builders.py`
  Здесь лежит сам характер, memory prompt и структура инструкций.
- `app/services/ai_service.py`
  Здесь prompt собирается с реальными данными чата.
- `app/services/memory_provider.py`
  Здесь prompt для памяти используется для извлечения summary + facts.

## На каком языке писать сам текст prompt

Для этого бота лучше:

- character prompt: русский;
- memory prompt: русский;
- section tags и техническая структура: можно оставлять на английском (`<identity>`, `<tool_policy>`, `<memory>`), это удобно и читабельно.

Почему русский:

- целевой чат русский;
- характер, дерзость, подколы, мягкость и границы в русском языке управляются точнее;
- снижается риск того, что модель будет звучать как переведенный саппорт.

## Правило редактирования

Если хочешь менять поведение:

- меняй `build_character_system_prompt()`, если речь про характер и стиль;
- меняй `build_memory_extraction_messages()`, если речь про память;
- не зашивай большие prompt-строки обратно в `ai_service.py`.
