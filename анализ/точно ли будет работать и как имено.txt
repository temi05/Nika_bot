Да, эта система точно будет работать, но есть несколько критических технических нюансов на стороне Telegram API, которые нужно учесть при написании кода, иначе логика сломается:
​Ограничение на отслеживание пригласительных ссылок: Отследить владельца по ссылке можно только в том случае, если именно ваш бот безопасности создал эту ссылку. Если ссылку сгенерирует другой администратор чата, то в событии входа chat_member сервер Telegram из соображений конфиденциальности пришлет обрезанную ссылку (ее конец будет заменен на многоточие ...) [1]. Из-за этого вы не сможете сопоставить ее со своей базой. Бот должен быть единственным генератором ссылок.
​Особенность публичных чатов: Если ваша группа публичная (имеет открытый юзернейм @group), отслеживание ссылок может давать сбои. Официальное приложение Telegram для Android иногда обрабатывает переход по ссылке в публичный чат как прямой вход, и поле invite_link в данных от API может просто отсутствовать ``. Архитектура с персональными ссылками идеально и безотказно работает именно в закрытых (Private) супергруппах.
​«Слепота» бота по умолчанию: Чтобы бот вообще видел, кто кого добавил или по какой ссылке перешел, при инициализации бота (настройке вебхука или пулинга) программист обязан жестко прописать массив allowed_updates, включив туда строку "chat_member" [2]. Если этого не сделать, Telegram просто не будет присылать боту уведомления о новых участниках.
​Реальность теневого бана: Метод restrictChatMember работает в точности так, как вам нужно. Вы можете бессрочно выставить параметры вроде can_send_messages и can_send_media_messages в False [3]. Юзербот останется висеть в списке участников, но любые попытки скрипта отправить спам будут отбиваться сервером.
​Тегирование владельца: Визуальная подсветка владельца (метод setChatMemberTag и поле sender_tag) — это самая свежая функция платформы, добавленная в Telegram Bot API 9.5 от 1 марта 2026 года [4]. Вам нужно убедиться, что фреймворк, на котором вы будете писать бота (например, aiogram для Python или Telegraf для Node.js), уже обновлен для поддержки этой новой версии API.
​Если ваш разработчик учтет эти ограничения при проектировании базы данных и доступов, система будет функционировать именно так, как задумано, оставляя злоумышленника в полном неведении.
