Я проанализировал вашу задачу и нашел способ реализовать такую систему защиты, используя продвинутые возможности Telegram Bot API. Вот краткий алгоритм того, что нужно сделать:
​1. Связывание зашедшего бота с его владельцем
Чтобы бот понимал, кто именно привел автоматизированные профили, ему нужно дать права администратора и обязательно подписать на получение обновлений типа chat_member.[1] При входе или добавлении пользователя Telegram генерирует событие ChatMemberUpdated. В этом объекте есть поле from_user, которое содержит ID участника, инициировавшего добавление.[2] Если ботов загоняют по пригласительным ссылкам, ваш бот должен отзывать общую ссылку и выдавать каждому участнику персональную уникальную ссылку. Так вы всегда будете знать, откуда произошла утечка.
​2. Распознавание юзерботов (без использования капчи)
Так как юзерботы умеют решать капчу, выявлять их нужно по метаданным и поведению:
​Возраст аккаунта: Telegram выдает ID последовательно. Можно использовать математическую интерполяцию для вычисления примерной даты регистрации аккаунта по его ID. Большинство юзерботов — это "новореги", созданные для одной атаки.
​Подписка Premium: Если у зашедшего профиля есть Telegram Premium, вероятность того, что это бот, стремится к нулю, так как это невыгодно экономически.
​Имена и био: Часто ботнеты используют бессмысленные наборы букв в логинах или пустые описания.
​Глобальные базы: При входе пользователя его ID нужно моментально проверять по открытым антиспам-базам (например, CAS API).
​3. Теневая блокировка (чтобы дезориентировать владельца)
Нельзя использовать стандартный метод удаления (banChatMember), так как владелец бота моментально увидит, что его профиль выкинули из группы, и сменит тактику.
Вместо этого нужно использовать "теневой бан" через метод restrictChatMember. Этим методом вы просто забираете у юзербота право отправлять любые сообщения и медиа. Для владельца его юзербот будет по-прежнему числиться в списке участников группы, но все попытки скрипта отправить спам будут выдавать скрытую ошибку на уровне API. Владелец будет впустую тратить ресурсы, не понимая, почему сообщений нет в чате.
​4. Подсветка владельца ботов
С 1 марта 2026 года в Telegram Bot API появилась возможность присваивать участникам кастомные теги через метод setChatMemberTag.[3] Когда ваша система свяжет юзербота с конкретным инвайтером, бот может автоматически повесить на владельца визуальный тег (например, "Владелец ботов"), который будет отображаться рядом с его именем прямо в чате. При этом самого владельца бот не забанит, но администраторы будут четко видеть его в интерфейсе. Дополнительно все инциденты и неизменный ID владельца пишутся в вашу закрытую реляционную базу данных для ведения логов.
